# Project Schedule

## Overview

The project schedule was created in **Microsoft Project** based on the Work Breakdown Structure (WBS). It defines the sequence of activities, task dependencies, project milestones, and resource assignments.

---

## Key Activities

- Requirements analysis
- Vendor selection
- System configuration
- Data migration
- System testing
- User training
- Go Live
- Project closure

---

## Scheduling Approach

The schedule uses task dependencies to ensure that each project phase is completed in the correct order. Milestones were added to monitor project progress and track key implementation stages.

The Critical Path was identified to highlight activities that directly impact the project completion date.
