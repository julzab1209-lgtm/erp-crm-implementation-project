# Project Planning

## Overview

The CRM implementation project was planned using **Microsoft Project**, following a structured approach commonly used in ERP implementation projects. The planning process covered project scope, scheduling, resource allocation, risk assessment, cost estimation, and project monitoring.

For a general description of the project, see the [Project Overview](project-overview.md).

---

## Work Breakdown Structure (WBS)

The project was divided into manageable phases to simplify planning and execution.

### Project Phases

1. Requirements Analysis
2. Vendor Selection
3. System Configuration
4. Data Migration
5. Testing
6. User Training
7. Go Live
8. Project Closure

The Work Breakdown Structure (WBS) provided a clear overview of project activities and helped organize tasks throughout the implementation lifecycle.

---

## Project Schedule

The project schedule was created in **Microsoft Project** using task dependencies and milestones.

Key scheduling activities included:

- defining project phases,
- assigning task durations,
- establishing task dependencies,
- identifying project milestones,
- monitoring project progress.

The schedule ensured that project activities were completed in the correct sequence while supporting efficient project coordination.

---

## Resource Planning

Project resources were assigned according to their responsibilities and availability.

### Project Roles

- Project Manager
- Business Analyst
- IT Administrator
- IT Specialist
- CRM Vendor
- End User

Resource allocation was analysed to balance workloads and identify potential overallocation before project execution.

---

## Risk Assessment

Potential implementation risks were identified during the planning phase to reduce the likelihood of project delays and operational issues.

Examples of identified risks include:

- project delays,
- budget overruns,
- data migration issues,
- technical integration problems,
- insufficient user training.

Early risk identification supports better decision-making and improves project success.

---

## Cost Analysis

Project costs were estimated based on the planned implementation activities and required resources.

Cost estimation helps to:

- support budgeting,
- monitor project expenses,
- improve financial planning,
- evaluate implementation feasibility.

---

## Critical Path

Critical Path analysis was used to identify the sequence of activities that directly affect the project completion date.

Understanding the Critical Path helps project managers:

- prioritize critical tasks,
- monitor project progress,
- minimize scheduling delays,
- improve implementation control.

---

## Summary

Effective project planning provides a structured foundation for successful ERP implementation. Combining scheduling, resource management, risk assessment, cost estimation, and Critical Path analysis improves project control and supports timely project delivery.

For details about the supported workflow, see the [ERP Business Process](erp-business-process.md).
